# План реализации ChatList

Поэтапный порядок действий по реализации программы на основании [PROJECT.md](PROJECT.md).

---

## Этап 1. Подготовка проекта

1. Обновить `requirements.txt`: PyQt6, httpx (или requests), python-dotenv.
2. Создать структуру модулей:
   - `db.py` — работа с SQLite;
   - `models.py` — логика нейросетей и отправки промтов;
   - `network.py` — HTTP-запросы к API;
   - `main.py` — графический интерфейс.
3. Добавить `.env.example` с примерами переменных для API-ключей (без реальных секретов).
4. Убедиться, что `.env` указан в `.gitignore`.

---

## Этап 2. База данных (`db.py`)

1. Реализовать подключение к SQLite и инициализацию схемы (см. [DATABASE.md](DATABASE.md)).
2. Добавить CRUD-операции для таблиц:
   - `prompts` — создание, чтение, поиск, удаление;
   - `models` — создание, чтение, обновление, активация/деактивация;
   - `results` — сохранение выбранных ответов, чтение истории;
   - `settings` — чтение и запись настроек.
3. Подготовить начальные данные: несколько моделей-заглушек или пустой список.
4. Проверить создание БД и базовые операции отдельным скриптом или тестовым вызовом.

---

## Этап 3. Сетевой слой (`network.py`)

1. Реализовать загрузку API-ключей из `.env` по имени переменной (`api_id` из таблицы `models`).
2. Создать базовую функцию отправки запроса к API (POST, таймаут, обработка ошибок).
3. Добавить адаптеры под разные типы API (OpenAI-совместимый формат как базовый).
4. Вернуть унифицированный результат: текст ответа или сообщение об ошибке.
5. Обеспечить асинхронность или фоновое выполнение, чтобы GUI не зависал при ожидании ответов.

---

## Этап 4. Логика моделей (`models.py`)

1. Получать список активных моделей из БД (`is_active = 1`).
2. Реализовать отправку одного промта во все активные модели.
3. Формировать временную структуру результатов в памяти:
   - название модели;
   - текст ответа;
   - флаг `selected` (по умолчанию `False`).
4. Реализовать сохранение отмеченных строк в таблицу `results`.
5. Реализовать очистку временной таблицы после сохранения или при новом запросе.

---

## Этап 5. Главное окно — ввод промта (`main.py`)

1. Заменить демо-интерфейс на окно ChatList.
2. Добавить поле ввода промта (многострочное).
3. Добавить выбор сохранённого промта из таблицы `prompts` (выпадающий список или отдельная панель).
4. Кнопки: «Отправить», «Сохранить промт» (опционально), «Очистить».
5. При выборе сохранённого промта — подставлять текст в поле ввода.

---

## Этап 6. Главное окно — таблица результатов

1. Добавить `QTableWidget` (или `QTableView` + модель) для временных результатов.
2. Колонки: чекбокс, модель, ответ.
3. Кнопка «Сохранить» — записывает строки с отмеченным чекбоксом в `results` и очищает таблицу.
4. При вводе нового промта и повторной отправке — полностью пересоздавать временную таблицу.
5. Показывать индикатор загрузки на время ожидания ответов от API.

---

## Этап 7. Управление моделями и настройками

1. Добавить окно или вкладку «Модели»:
   - список моделей;
   - добавление, редактирование, удаление;
   - переключатель `is_active`.
2. Поля формы: имя, URL API, имя переменной окружения (`api_id`).
3. Добавить окно «Настройки» для таблицы `settings` (при необходимости).
4. Добавить окно «История» для просмотра сохранённых `results`.

---

## Этап 8. Дополнительные функции (опционально)

1. Поиск и сортировка во всех таблицах интерфейса.
2. Экспорт выбранных результатов в Markdown и JSON.
3. Расширенная поддержка провайдеров: OpenAI, DeepSeek, Groq и др.
4. Логирование запросов в файл или таблицу.
5. Теги для промтов: ввод, фильтрация, отображение.

---

## Этап 9. Финализация

1. Проверить полный сценарий:
   - ввод промта → отправка → просмотр ответов → сохранение выбранных → новый запрос.
2. Обработать краевые случаи: нет активных моделей, нет API-ключа, ошибка сети, пустой промт.
3. Обновить `README.md` с инструкцией по установке и настройке `.env`.
4. При необходимости собрать исполняемый файл через PyInstaller.

---

## Порядок разработки (кратко)

```
Подготовка → db.py → network.py → models.py → main.py (ввод) → main.py (результаты) → управление → опции → тестирование
```

Каждый этап завершается проверкой работоспособности перед переходом к следующему.
